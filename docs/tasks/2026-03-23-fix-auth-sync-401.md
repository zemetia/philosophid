# Task: Fix 401 Unauthorized on /api/auth/sync (JWT Token Implementation via HttpOnly Cookies)

- **Date**: 2026-03-24
- **Status**: Testing Phase
- **Source**: docs/tasks/23-03-2026.md (Task #3)

## 🎯 Goal

Resolve the 401 Unauthorized error when calling `/api/auth/sync` using best security practices by:
1. Generating a custom JWT containing `user_id` and `email` upon the initial successful sync after login (both manual and Firebase).
2. Storing the Custom JWT securely inside an **HttpOnly, Secure cookie** (`philosophid_session`), mitigating XSS vulnerabilities compared to `localStorage`.
3. Decoupling subsequent API requests (like completing registration) from the Firebase Client SDK because the browser automatically attaches the HttpOnly cookie.
4. Making the JWT expiration time configurable via an environment variable in `.env`.

This architectural shift provides seamless SSR support and prevents 401 errors caused by Firebase Client Auth State desyncs.
