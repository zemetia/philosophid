## 🎯 Goal

Implement a secure backend authentication flow using Firebase Admin SDK, a robust user registration/sync mechanism, and a comprehensive dashboard API to support the user landing experience.

## 📋 Implementation Checklist

### Phase 1: Authentication Infrastructure
- [x] **Step 1.1: Firebase Admin Configuration**
  - [x] Create/Update `src/lib/firebase-admin.ts` to initialize Firebase Admin using environment variables.
  - [x] Ensure `FIREBASE_PROJECT_ID`, `FIREBASE_CLIENT_EMAIL`, and `FIREBASE_PRIVATE_KEY` are handled.
- [x] **Step 1.2: Token Verification Logic**
  - [x] Create `src/lib/auth-server.ts` containing `getAuthUser()` helper.
  - [x] This helper should:
    1. Extract the `Authorization: Bearer <token>` from headers.
    2. Verify the token using `firebase-admin`.
    3. Return the Firebase user object or throw an unauthorized error.

### Phase 2: Registration & Sync Logic
- [x] **Step 2.1: Implement Sync User Service**
  - [x] Add `syncUserWithDatabase(firebaseUser: any)` to `src/lib/services/user-service.ts`.
  - [x] Logic: `upsert` user in Prisma based on `firebaseUid`.
  - [x] Map `email`, `displayName` (to `name`), and `photoURL` (to `avatarUrl`).
- [x] **Step 2.2: Create Sync API Endpoint**
  - [x] Create `src/app/api/auth/sync/route.ts`.
  - [x] Method: `POST`.
  - [x] Action: Call `getAuthUser()` then `syncUserWithDatabase()`.
  - [x] Returns user data from DB.

### Phase 3: Dashboard API Implementation
- [x] **Step 3.1: Define Dashboard Data Requirements**
  - [x] Need: Profile summary (name, role, score).
  - [x] Need: Statistics (total papers, draft count, published count).
  - [x] Need: Recent Activity (last 3-5 papers).
  - [x] Need: Badge highlights.
- [x] **Step 3.2: Implement Dashboard Service**
  - [x] Create `getUserDashboard(userId: string)` in `UserService`.
- [x] **Step 3.3: Create Dashboard API Endpoint**
  - [x] Create `src/app/api/user/dashboard/route.ts`.
  - [x] Method: `GET`.
  - [x] Security: Protected via `getAuthUser()`.

### Phase 4: Verification & Hardening
- [x] **Step 4.1: Error Handling**
  - [x] Ensure 401 Unauthorized is returned for invalid tokens.
  - [x] Ensure 500 Internal Server Error is handled gracefully.
- [x] **Step 4.2: Performance Audit**
  - [x] Use `Promise.all` for dashboard data fetching to minimize latency.
